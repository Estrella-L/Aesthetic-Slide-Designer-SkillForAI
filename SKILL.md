# Aesthetic Slide Designer

把任意视觉参考（图片或网页）的审美风格，转化为一套风格统一、可演示的 HTML 幻灯片。

## 何时使用

- 用户给了一张图片（设计稿、海报、UI 截图、品牌物料）说"按这个风格做 slide"。
- 用户给了一个网页 URL 或 HTML 文件，说"参照这个网站的审美做演示页"。
- 用户想要把内容做成与某个参考一致风格的 PPT 风格 HTML 页面。

## 总体流程

按顺序执行：**0) 查预设 → 1) 采集参考 → 2) 提取风格 → 3) 确认内容与结构 → 4) 生成 slide → 5) 自检 → 6) 沉淀预设**。

### 0. 查预设（复用已有风格）

- 先读 `presets/index.md`，看是否已有用户想要的风格。
- 如果用户说"用上次那个 X 风格"或描述的风格能对上某个预设，直接读 `presets/<name>.md` 拿到现成的 Style Spec，**跳过第 1-2 步**，进入第 3 步。
- 如果没有匹配的预设，才走完整提取流程。

### 1. 采集参考

- **图片输入**：直接观察用户附带的图片。如果是本地路径，读取后分析。
- **网页输入**：
  - 有 URL 时，用 web_fetch 获取 HTML（必要时用 `rendered` 模式处理 JS 重的页面），从中读取 CSS、配色、字体声明。
  - 有本地 HTML 文件时直接读取文件及其引用的 CSS。
- 如果两者都没有，向用户索要参考，或让其用一两个形容词描述目标风格。

### 2. 提取风格

参照 `references/style-extraction.md` 系统地提取这些维度，并整理成一个"风格规格(Style Spec)"：

- **配色**：主色、辅助色、强调色、背景色、文字色（尽量给出 HEX）。
- **字体**：标题/正文字族、字重、字号层级、字间距、行高。
- **布局**：网格、留白密度、对齐方式、内容密度（紧凑 vs 疏朗）。
- **情绪基调**：例如 极简/科技/复古/手作/奢华/活泼。
- **细节语言**：圆角、阴影、边框、分隔线、图标风格、渐变、纹理。

把 Style Spec 用一小段总结呈现给用户确认，再继续。

### 3. 确认内容与结构

- 明确这套 slide 要讲什么内容、几页、每页要点。
- 若用户只给了风格没给内容，先索取内容大纲或用占位内容并说明。
- 决定每页类型：封面页 / 章节页 / 要点页 / 图文页 / 数据页 / 结尾页。

### 4. 生成 slide

- 以 `assets/slide-template.html` 为骨架，把 Style Spec 落到 CSS 变量里（`:root` 中的 `--color-*`、`--font-*` 等）。
- 关键规则见 `references/slide-templates.md`：
  - 所有 slide 共用一套设计 token，保证整体统一。
  - 每页是一个 16:9 的 `.slide` 容器，支持键盘左右键翻页。
  - 单文件输出（CSS 内联在 `<style>`，必要字体用 Google Fonts 链接），方便直接打开和分享。
  - 响应式、可打印（`@media print` 每页一张）。
- 默认输出到项目里的 `slides.html`（或用户指定文件名）。

### 5. 自检

生成后逐项核对：
- 配色/字体/间距是否确实贴合参考，而非套用通用模板。
- 在浏览器中能打开、翻页正常、无溢出/错位。
- 文字对比度足够（无障碍），图片有 alt。
- 把最终风格与参考做一句话对比说明给用户。

### 6. 沉淀预设（保存复用）

只要这次是**新提取**的风格（而非复用已有预设），就主动把它存下来：

- 按 `references/preset-format.md` 的格式，把 Style Spec + CSS 变量写成一个文件 `presets/<kebab-name>.md`。
- 名字用简短易记的 kebab-case，例如 `apple-keynote`、`xiaohongshu-warm`、`cyberpunk-neon`。
- 在 `presets/index.md` 表格里追加一行（名字、一句话描述、关键词），方便下次检索。
- 如果用户对已有预设做了调整，更新对应文件而不是新建。
- 存完告诉用户：「已保存为预设 `<name>`，下次说『用 <name> 风格』即可直接复用。」

> 注意：保存依赖文件写入权限。在 Claude Code / Kiro 等本地环境可长期保留；纯 Claude.ai 沙箱内可能不持久，此时提示用户手动下载预设文件留存。

## 注意

- 复刻"审美风格"，不是抄袭具体内容或受版权保护的素材。
- 如果参考风格信息不足（如只有一张低清图），明确说明你的推断假设。
- 优先用 CSS 变量驱动风格，方便用户后续一处改全局调。
